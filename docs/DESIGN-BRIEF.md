# בריף עיצוב (UX/UI) ודרישות פיתוח — Tehillim Campaign Manager

מסמך זה מיועד למסירה למעצב/ת UX/UI. הוא מרכז את **כל העמודים, המסכים, הקומפוננטים והפונקציות** שדורשים עיצוב, יחד עם **דרישות נגישות** (ת"י 5568 / WCAG 2.0 AA) ודרישות פיתוח להעברה נקייה למפתח.

> הקהל הוא **דובר עברית, מובייל-פירסט, ומגיע ברובו מוואטסאפ**. לכן: RTL מלא, טיפוגרפיה עברית נדיבה, מסכים שעובדים מצוין בנייד, וכל מסך חייב להיראות טוב גם כשמגיעים אליו מקישור בוואטסאפ.

---

## 0. עקרונות-על

1. **מרחב קדוש מול מרחב מסחרי** — מסך קריאת הפרק/התפילה (טקסט התהילים) הוא **מרחב מוגן ללא פרסום ובלי הסחות**. פרסום מותר רק במרחבים המסחריים (ארכיון, ראש קמפיין, לפני/אחרי קריאה, מסכי תודה, סיידברים, פוטר מייל). ראו §6.
2. **מובייל-פירסט** — לעצב קודם ל‑375–414px, אחר כך דסקטופ. נקודת שבירה עיקרית קיימת: 768px.
3. **נגישות מובנית** — לא שכבה בסוף. כל קומפוננט מעוצב עם מצבי focus, ניגודיות ותוויות. ראו §2.
4. **עיצוב מבוסס Design Tokens** — המערכת מאפשרת למנהל לשנות צבעים/רדיוס/פונט מלוח הבקרה. העיצוב חייב לעבוד דרך מערכת ה‑tokens (§1), לא צבעים קשיחים.

---

## 1. שפה עיצובית ו‑Design Tokens

יש לעצב **Design System** עם הטוקנים הבאים (כבר נשלטים מלוח הבקרה — לעצב סביבם ולספק ערכי ברירת מחדל נגישים):

| טוקן | תיאור |
|------|-------|
| `primary` | צבע ראשי לכפתורים |
| `secondary` | צבע משני / פס התקדמות |
| `card_bg` · `page_bg` · `dashboard_bg` | רקעים |
| `text` · `muted` | צבעי טקסט ראשי/משני |
| `field_bg` · `field_border` | שדות טופס |
| `button_text` | טקסט בכפתור |
| `radius` · `button_radius` · `field_radius` | עיגול פינות (כרטיס/כפתור/שדה) |
| `title_size` · `font_family` · `max_width` | טיפוגרפיה ופריסה |
| `button_padding_x/y` | ריווח כפתור |

**תוצרים נדרשים:** ערכת צבעים נגישה (ברירת מחדל + מצב כהה אופציונלי), סולם טיפוגרפי עברי, סולם ריווח (spacing scale), סולם רדיוסים, סגנונות צל, ומערכת אייקונים אחידה (כולל אייקוני סטטוס פרק — ראו §2).

## 2. דרישות נגישות (חובה — ת"י 5568 / WCAG 2.0 AA)

צ'קליסט מחייב לכל מסך וקומפוננט:

- **RTL מלא** + `lang="he"`. בדיקת זרימה דו-כיוונית (טקסט עברי + קישורים/מספרים באנגלית).
- **אסור צבע כאמצעי מידע יחיד** — סטטוס הפרק (פנוי/נתפס/הושלם) חייב **אייקון + טקסט**, לא רק צבע. לעצב 3 אייקוני סטטוס ברורים.
- **ניגודיות:** טקסט רגיל ≥ 4.5:1, טקסט גדול/אלמנטים ≥ 3:1. לבדוק גם את צבעי ברירת המחדל של הכפתורים (טקסט לבן על צבע משתנה).
- **תוויות טופס** משויכות (`for`/`id`), סימון שדה חובה/רשות באופן ברור (לא רק צבע), והודעות שגיאה צמודות לשדה.
- **פוקוס נראה** בכל אלמנט אינטראקטיבי (outline ברור), וניהול פוקוס: בכניסה למסך הקריאה הפוקוס עובר לכותרת הפרק.
- **יעדי מגע ≥ 44×44px** (קיים כבר min-height 52–62px בשדות — לשמר).
- **פס התקדמות** מעוצב כך שיתמוך ב‑`role="progressbar"` + ערך מספרי גלוי (לא רק ויזואלי).
- **תפריט בחירה ארוך (פרקים):** קבוצות `optgroup` (לא קווים מפרידים), קריא במובייל.
- **הודעות דינמיות** (העתקת קישור → "הועתק") עם משוב נגיש (`aria-live`).
- **אנימציות** מכבדות `prefers-reduced-motion`.
- **אייקונים/אימוג'י** עם חלופה טקסטואלית.
- **פרסומות** מתויגות "בחסות/פרסומת", נגישות למקלדת, לא לוכדות פוקוס.
- היררכיית כותרות תקינה (H1→H2→H3), טקסט חלופי לתמונות, ו"דלג לתוכן".

## 3. מסכי פרונט (Front-End) לעיצוב

לכל מסך: מטרה · קומפוננטים · מצבים · אזורי פרסום · הערות נגישות.

### 3.1 ⭐ עמוד קמפיין (Single) — המסך המרכזי
- **קומפוננטים:** כותרת/הקדשה, תיאור, **פס התקדמות**, בלוק סטטיסטיקות (אחוז, יעד בסיס, בונוס, ספר נוכחי, פרקים שהושלמו), **טופס הצטרפות** (§3.3), **רשת הפרקים** (§3.4), **בלוק שגריר** (§3.9), בלוק שיתוף (§3.10).
- **מצבים:** קמפיין פעיל / מושהה / הושלם; "נוצר בהצלחה" (notice + שיתוף); הודעות `tcm_msg` (פרק נתפס/אין פנוי/הושלם הכל).
- **פרסום:** `campaign_header` (מתחת לכותרת), `before_join`, `after_join`. **לא** ברשת הפרקים בזמן בחירה.

### 3.2 ⭐ ארכיון קמפיינים `[tehillim_campaigns]`
- **קומפוננטים:** רשת כרטיסי קמפיין (כותרת, "חלוקת תהילים", פס התקדמות, יעד/בונוס, כפתור הצטרפות).
- **מצבים:** ריק (אין קמפיינים), טעינה, הרבה קמפיינים (גלילה/עימוד).
- **פרסום:** `archive_top`, `archive_inline` (כרטיס פרסום בין כרטיסי קמפיין — לעצב **כרטיס native** תואם).

### 3.3 ⭐ טופס הצטרפות `[tehillim_join_form]`
- **קומפוננטים:** בורר פרק (אוטומטי / פרק ספציפי / "קח X פרקים" / "ספר שלם"), שדות שם/אימייל/טלפון (רשות), הערת פרטיות, ווידג'ט Turnstile, כפתור שליחה גדול.
- **מצבים:** רגיל, אין פרקים פנויים, שגיאת אימות, נשלח. ולידציה inline.
- **נגישות:** קריטי — תוויות משויכות, `optgroup`, סימון רשות ברור, הודעות שגיאה צמודות.

### 3.4 רשת מצב הפרקים `[tehillim_chapters]`
- **קומפוננטים:** גריד 150 פרקים עם 3 סטטוסים. מקרא (legend).
- **נגישות:** ⭐ **אייקון+טקסט לכל סטטוס** (לא רק צבע). תוויות נגישות לכל אריח.

### 3.5 ⭐ מסך קריאת פרק (Reader) — מרחב קדוש
- **קומפוננטים:** כותרת "הפרק שלך — תהילים פרק X", מחוון התקדמות אישי ("פרק 2 מתוך 5"), רשימת הפרקים שנלקחו, **טקסט הפרק** (טיפוגרפיה גדולה וקריאה במיוחד), כפתורי "סיימתי את הפרק" / "סיימתי ורוצה עוד".
- **מצבים:** פרק יחיד / כמה פרקים / טקסט פרק חסר / קישור לא תקין.
- **פרסום:** ❌ **אסור בתוך המסך הזה.** מותר רק `pre_reading` (לפני) ו‑`post_done` (אחרי סימום).
- **נגישות:** טיפוגרפיה לקריאה ממושכת (גודל, line-height, רוחב שורה), פוקוס לכותרת בכניסה, ניגודיות גבוהה.

### 3.6 טופס יצירת קמפיין `[tehillim_create_campaign_form]`
- שדות: הקדשה/כותרת, יעד ספרים, תיאור. מצבי: ריק, נשלח, שגיאה.

### 3.7 ⭐ "הקמפיינים שלי" / ניהול בעלים `[tehillim_my_campaigns]`
- **קומפוננטים:** כרטיס לכל קמפיין + פאנל ניהול: עריכת כותרת/תיאור/יעד, הוספת ספר בונוס, **טופס הכנת מייל למשתתפים** (ממתין לאישור), בלוק שיתוף.
- **מצבים:** לא מחובר, אין קמפיינים, הודעת הצלחה.

### 3.8 "הפעילות שלי" `[tehillim_my_activity]`
- טבלת פעילות אישית (קמפיין, פרק, סטטוס, פעולה). מצב ריק / לא מחובר.

### 3.9 בלוק שגריר `[tehillim_ambassador_invite]` ודאשבורד `[tehillim_ambassador_dashboard]`
- **קומפוננטים:** קישור שגריר אישי + העתקה + שיתוף וואטסאפ; דאשבורד עם מיני-סטטיסטיקות (נלקחו/הושלמו/ספרים), ו‑**לוח מובילים** (חדש, §10 בתכנית).
- מצבים: לא מחובר, אין קישורי שגריר.

### 3.10 קומפוננטים/וידג'טים קטנים (לאלמנטור)
פס התקדמות, אחוז, מספר משתתפים, פרקים נותרים, ספרים שהושלמו, **בלוק סטטיסטיקות**, כפתור CTA דינמי, **תגית דחיפות** ("נשארו רק X"), בלוק שיתוף, וכן `[tehillim_data]` (נתון בודד חופשי). לעצב כ‑**design system של widgets** עקבי.

### 3.11 ⭐ טופס הרשמה לרשימות `[tehillim_subscribe]` (חדש, §11)
- בחירת ערוץ (מייל / **וואטסאפ**), שדה טלפון, תיבת הסכמה, כפתור. + עמוד "תודה על ההרשמה" ועמוד ניהול/ביטול מנוי.

### 3.12 ⭐ אזור סגולות ותפילות (חדש, §12)
- ארכיון מסונן לפי קטגוריה + חיפוש, כרטיס תפילה, **עמוד תפילה בודד** (מרחב קדוש לקריאה — בלי פרסום על הטקסט), כפתורי "קבל תפילה יומית" / "צרף לקמפיין".

### 3.13 קומפוננטים גלובליים ומצבים
הודעות (notice הצלחה/שגיאה/אזהרה), מצבי ריק, מצבי טעינה, מצבי שגיאת קישור, דיאלוג אישור (במקום `confirm` נוכחי), ופוטר/מיתוג עדין.

## 4. מסכי ניהול (Admin / wp-admin) לעיצוב

עיצוב בהתאמה ל‑WordPress admin אך מסודר, RTL ונגיש:
1. **דאשבורד ניהול חלוקות** — טבלת קמפיינים + פעולות (בונוס/איפוס).
2. **הגדרות** (גדול מאוד) — לפצל ל**טאבים**: קישורים, חוויית משתמש, וובהוקים, מיילים, תזכורות/שחרור, **רשימות/וואטסאפ** (חדש), **פרסום** (חדש), עיצוב.
3. **עורך 150 הפרקים** — UI מתקפל/מחפש (כיום 150 שדות) — לשפר UX.
4. **הודעות לאישור** — תור אישור הודעות בעלים.
5. **לוג פעולות** — טבלה עם עימוד/סינון (חדש).
6. **מדריך שימוש ושורטקודים**.
7. **מטה-בוקס בעריכת קמפיין** — הגדרות יעד/בונוס/סטטוס.
8. **חדש:** ניהול **פרסומות** (§9), ניהול **רשימות ונרשמים** (§11), CPT **סגולות** (§12).

## 5. תבניות הודעות (מייל + וואטסאפ)

לעצב מערכת תבניות עקבית (HTML מייל RTL + פורמט וואטסאפ):
קבלת פרק · קבלת כמה פרקים · קבלת ספר שלם · תזכורת · אזהרת שחרור · השלמת ספר · השלמת ספר ע"י משתתף · אישור פתיחת קמפיין · הודעת בעלים למשתתפים · **תוכן יומי לרשימה** · תודה על הרשמה.
- וואטסאפ: פורמט קצר, תבניות מאושרות (HSM), CTA אחד ברור.

## 6. מפת אזורי פרסום (Ad Zones) — היכן מותר

| אזור | מסך | מותר? |
|------|-----|-------|
| `archive_top` · `archive_inline` | ארכיון | ✅ |
| `campaign_header` | ראש עמוד קמפיין | ✅ |
| `before_join` · `after_join` | סביב טופס הצטרפות | ✅ |
| `pre_reading` | לפני כניסה לקריאה | ✅ |
| `post_done` · `completion` | אחרי סימון/השלמה | ✅ |
| `share_screen` · `sidebar` · `email_footer` | שיתוף/צד/מייל | ✅ |
| **בתוך טקסט הפרק/התפילה** | Reader | ❌ אסור |

פורמטים לעצב: באנר רספונסיבי, קישור טקסט, **כרטיס native**, רצועת חסות. כולם עם תיוג "בחסות".

## 7. דרישות פיתוח / מסירה (Handoff)

- **תוצרים מהמעצב/ת:** קבצי Figma עם Design System (tokens, components, states), מובייל + דסקטופ לכל מסך §3–§4, מצבי ריק/שגיאה/טעינה/הצלחה, ערכת אייקונים (כולל 3 סטטוסי פרק), ותיעוד ניגודיות.
- **RTL ועברית** בכל המסכים, כולל טקסט ארוך (תיאורי קמפיין, טקסט פרק).
- **התאמה ל‑tokens** של §1 כדי שלוח הבקרה ימשיך לשלוט בעיצוב.
- **רספונסיביות** מלאה (375 → 1280+), נקודת שבירה 768px לפחות.
- **נגישות (§2)** מתועדת לכל קומפוננט (focus, ARIA, ניגודיות).
- **ללא תלות בתמונות כבדות** במרחב הקריאה (ביצועים).
- אספקת assets (SVG אייקונים, לוגו, רקעים) בפורמטים מותאמי-וב.
